Distributed Instant Messaging (DIM) Protocol

4 minute read

Packets

Packets are in a (hopefully) incredibly simple and intuitve format.

<Version> SP <Details> LF
[Header Flags] LF
"---" LF
<Message>
["---" OR "..."] LF

Now for the components.

SP     = " " / 0x20
LF     = 0x0A

Version     = "dim" "/" <Major> "." <Minor>
Details     = "[Request | Response Code] SP [DIM URI | Details]"
Header Flag = <Key> "=" <Value> ";"
Message     = Any valid Unicode string where the sequences "---", "...", and "&" are properly encoded.

Encoding: 
"---" Gets Encoded As "&hr;"
"..." Gets Encoded As "&cont;"
"&"   Gets Encoded As "&amp;"

Valid and Invalid Packet Examples

This packet is INVALID because there is a space after the semicolon on the seccond line.

dim/1.0 GET dim:user:user@domain.tld
encryption="aes"; do_not_track=true;
---
This is my content. I can put anything here
it can even span multiple lines!

The only thing that isn't allowed is tripple-hyphens must be encoded with 
the entity `&hr;`, and 3 dots must be encoded with `&cont;`

Other than that this is completely safe for any content!
---

To make this valid, simply remove the space.

dim/1.0 GET dim:user:user@domain.tld
encryption="aes";do_not_track=true;
---
This is my content. I can put anything here
it can even span multiple lines!

The only thing that isn't allowed is tripple-hyphens must be encoded with 
the entity `&hr;`, and 3 dots must be encoded with `&cont;`

Other than that this is completely safe for any content!
---

This packet is INVALID because the secconmd "..." incdicates the payload continues with another linked message, but no message is found.

dim/1.0 GET dim:channel:group_id@instance.domain.tld/channel
encryption="aes"; do_not_track=true;
---
Message. Anything except the control sequences of 3 hyphens (deliminator), 
and 3 dots (more transmissions in this packet, for network efficentcy), 
is allowed! This block ends with either 3 dots or 3 hyphens
...
dim/1.0 GET dim:channel:group_id@instance.domain.tld/rules
---
Message. Anything except the control sequences of 3 hyphens (end of transmission), and 3 dots (more transmissions in this packet, for network efficentcy), is allowed! This block ends with either 3 dots or 3 hyphens
...

To make it VALID, convert the seccond "..." into a "---".

dim/1.0 GET dim:channel:group_id@instance.domain.tld/channel
encryption="aes"; do_not_track=true;
---
Message. Anything except the control sequences of 3 hyphens (deliminator), and 3 dots (more transmissions in this packet, for network efficentcy), is allowed! This block ends with either 3 dots or 3 hyphens
...
dim/1.0 GET dim:channel:group_id@instance.domain.tld/rules
---
Message. Anything except the control sequences of 3 hyphens (end of transmission), and 3 dots (more transmissions in this packet, for network efficentcy), is allowed! This block ends with either 3 dots or 3 hyphens
